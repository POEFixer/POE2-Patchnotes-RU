# Как помочь с переводом

Спасибо за интерес! Любые правки, уточнения и предложения по терминологии приветствуются.

## Найти ошибку или неточность

1. Откройте Issue с темой `[<версия>/<секция>] краткое описание`. Пример: `[0.5.0/skills] Volatile Dead — неверный множитель`.
2. Приложите цитату из RU-перевода и оригинал EN, если возможно.

## Прислать правку

1. Сделайте fork репозитория.
2. Создайте ветку: `git checkout -b fix/<short-description>`.
3. Внесите правки. Соблюдайте правила из [STYLE.md](./STYLE.md) и термины из [GLOSSARY.md](./GLOSSARY.md).
4. Откройте Pull Request с описанием: что меняли и почему.

## Предложить термин в глоссарий

Если в новом патчноуте появился термин, которого нет в [GLOSSARY.md](./GLOSSARY.md):
1. Откройте Issue с темой `[glossary] <термин>`.
2. Укажите: оригинал EN, предлагаемый RU-вариант (или `keep`), категория (Skill / Item / Mechanic / …), контекст.

## Стиль перевода

- Прочитайте [STYLE.md](./STYLE.md) перед отправкой правок.
- Если правило не покрывает ваш случай — пишите в Issue, обсудим и дополним стиль-гайд.

## Что НЕ принимается

- «Художественные» вольные пересказы. Этот репо — про точный перевод патчноутов, а не про адаптацию.
- Изменение перевода устоявшегося глоссарного термина без обсуждения в Issue.
- Перевод имён собственных, которые в глоссарии помечены как `keep` (имена скиллов, предметов, ascendancy и т.д.).
